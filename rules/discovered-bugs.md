# Discovered Bugs

作業中に、現在のタスクとは無関係の既存バグを発見した場合:

1. 以下を**すべて満たす**（AND）こと: (a) **再現手順を書ける**（再現性がある）、(b) **期待動作と実動作の乖離をコード・テスト・一次仕様で示せる**（明確な不具合。仕様の曖昧さ・改善提案は対象外）、(c) **現在のタスクの変更が原因ではない既存の不具合**である（本筋の変更で生じた regression は本フローでなく本筋で直す）
2. **セキュリティ影響がある場合は issue を作らず先にユーザーへ報告する**（issue は repo の collaborator 全員に見えるため、精査前の脆弱性詳細をそのまま issue に書かない）。このケースは以降の issue 化ステップに進まない
3. セキュリティ影響が無ければ GitHub Issues で既存 issue を検索し、重複がなければ `gh issue create` で新規作成する。重複がある場合は発見の経緯をコメントで追記する
4. issue には以下を含める:
   - 再現手順
   - 期待される動作と実際の動作
   - 発見の経緯（PR 番号等。issue にセッション ID 等の内部識別子は含めない）
5. 現在のタスクはブロックせず続行する
